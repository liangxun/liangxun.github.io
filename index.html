变量类型与作用域

存储位置：

栈区（有时候也叫堆栈区）

常量区 （存储常量）

堆区（运行过程中动态创建申请的内存空间，需要在程序中释放）

寄存器中（这个用得比较少）

（问题：全局常量存储位置在哪里？ 静态区还是常量区）

可以通过关键字来指定变量的存储位置或者作用域

register 存储在寄存器中，访问速度最快

auto 存储在栈中。默认就是auto，所以基本没用过。【在c++11之后，auto关键字有新的含义，自动推测类型】

extern extern关键字让编译器知道不需要重新定义这个变量,这个其他文件定义过的全局变量

static static表示变量的作用域为当前文件。

注意区分static和const

static用来控制作用域；而const用来说明是否可以更改。这是两个完全不同的维度。

全局变量：在整个工程文件中有效

静态全局变量：限制在当前文件中有效，一般来说静态全局变量的数据安全性优于全局变量。

静态局部变量：在函数内部定义的static变量，在第一次调用函数的时候初始化，之后调用函数时候会自动跳过这一行。

const的作用

const修饰的标识符称为常量

  1. 值代替。下面两个功能一样，但是用const可以明确类型，更加安全。建议用const取代#define的值代替。

    C++中默认const是内部链接，即作用范围为当前文件，可以通过extern来扩展为外部链接

  2. 修饰指针
    有两种情况，指向const的指针；指针本身是const类型的。

  3. 作为函数参数，通常配合指针或引用来使用。

  4. 类中的const

    const类型成员变量在构造函数的初始化列表中初始化。const必须在进入函数体之前初始化。

    const类型的成员函数：表示这个成员函数不会修改成员数据。

指针与引用

引用和指针的区别？为什么提倡用引用而不是指针？

其实在底层引用是通过指针来实现的。引用可以看做是在指针之上做了一些限制条件。引用在声明的同时必须初始化，并且不能改变引用指向的对象。你如说是对象A的引用，那你不能说我再去作为B的引用，因为引用是非独立存在的。指针是可以独立存在的实体，指针可以的声明和初始化可以分开，并且指针的指向可以随便改动。所以说引用不会存在野指针那样的问题，更加安全一些。

主要用途：传参

函数的形参为指针或者引用。都是属于地址传递，地址传参的好处是不用大量复制变量，能够起到返回多个结果的效果。

  1. 为什么会需要动态内存？
  2. 使用流程

通过指针动态内存管理存在两个问题：

  1. 内存溢出。忘记释放指针所指内存；
  2. 引用非法内存的指针。我们释放了尚有指针引用的内存。

标准库<memory>头文件中提供了两种智能指针，可以自动释放所指向的对象：

  1. share_ptr允许多个指针指向同一个对象。会维持一个引用计数，计算器变为0则会自动释放所管理的内存

  2. unique_ptr 一个指针只能指向一个对象。比较常用的是从函数返回一个unique_ptr.

普通构造函数能被隐式调用，而explicit构造函数只能被显示调用。explicit只能用于构造函数这个地方。只需要在构造函数申明的地方说明就好。

封装。 访问控制机制

通过访问权限控制的成员可见性是封装的核心。

具有三种对成员的访问权限：

public 外部可见。对外接口

private 内部可见。不能从外部直接访问，如果有访问需求的话必须通过成员函数等方式。

protected 对家族内部可见。从当前类的外部来看，访问权限和private一样。 差别体现在继承的时候，继承类可以直接访问把基类的protected部分成员。

二、继承，多态

在类与对象级别考虑，代码重用有两种方式实现： 组合(composition)和继承(inheitance)。组合其实就是把当成数据类型来用，没啥好说的 。

子类的构造函数没有权利访问基类的私有数据成员，因此不能直接对基类的成员变量做初始化。因此需要初始化列表来完成基类的初始化，初始化列表是在进入函数体之前完成的。

继承方式：

默认是private

public继承，用得最多，使得基类的接口也是派生类的接口

protected继承，没怎么用过

向上类型转换？ upcasting

继承最重要的方面：基类与派生类的关系。派生类属于基类的类型。

向上类型转换总是是安全的，因为是从专门到一般的转换，即基类可能存在的成员函数派生类肯定都会有。所以编译器允许向上类型转换而不需要显示地说明。

基础简单的向上类型转换存在一个问题，向上转换成基类之后就只能是当做基类来处理。虚函数就是为了处理这个问题。

虚函数

如果不用虚函数，就等于还不懂得面向对象程序设计。在成员函数前面加上virtual关键字，那么这个成员函数就是虚函数了。

  1. 只需要在函数声明时候使用virtual，函数定义的地方不需要；
  2. 只要在基类中把函数声明为虚函数，派生类相应的函数也都是虚函数，不需要再用virtual显示说明，（说了也没有语法错误）。

编译器如何实现晚捆绑？

参考：《C++编程思想》p360讲解得非常清楚

编译器对每个包含虚函数的类创建一个表，通常称为VTABLE，并且会额外放置一个指针（加vpointer或者VPTR）来指向这个VTABLE.

在VTABLE中，放置了所有已声明为vitural的函数的地址。对于vitrual funx();，如果派生类B中重定义了func()，则VTABLE中存放B::func()的地址，若派生类没有重定义，则VTABLE中存放A::func().

每个类的VTABLE里面就存放了实际上会调用到的函数。从而实现了A类型的指针调用的函数并不一定是A::xxx()， 也可能是B::xxx()。其实就是把绝对地址引用改成了相对地址引用。

一个具有虚函数的类对应一个VTABLE，而每个该类类型的对象具有一个vpointer

抽象类， 纯虚函数

只要有一个纯虚函数，那么这个类就是抽象类，抽象类只能作为接口使用，不能实例化为对象。

抽象类A的VTABLE是不完全的，如果试图创建这个类的对象话，编译器会报错。

注意，在抽象类中，虚函数通常只什么，不用定义。但是如果要在这里定义也是可以的，因为可以从派生类中调用基类(基类是抽象类)的纯虚函数，其实可以放一些公共代码在这里。

Tensor什么时候共享内存？

万字综述，核心开发者全面解读PyTorch内部机制

进入Tensor的世界。从list或者 numpy array 构造Tensor的时候默认会是从新开辟内存的。
b = torch.Tensor(a) 那么a和b是分开存储的；a是numpy.array类型
c = torch.from_numpy(a) 这种方式a和c是共享内存的；

在Tensor的世界中
为了节省内存，tensor会尽可能地共享存储，用引用的方式，python语法本身也是这样的。但是记住共享内存的必要条件是两个变量存储的总各个要是相同的，或者切片。
共享底层存储的情况：

  1. 改变Tensor维度。 比如torch.squeeze(),torch.unsqeeze(), tensor.view()
  2. 直接赋值。 b = a
  3. 切片操作。 b = a[:5]
  4. 截断梯度。 b = a.detach()

开辟新的底层存储通常都需要手动指定：

  1. b = a.clone() b和a有不同的内存，但是pytorch把这种运算看做是计算图的一部分.可以用b = a.clone().detach() 来获得一个拥有新内存，也从计算图中分离出去的tensor.

离开Tensor的世界
b = a.item() get a Python number from a tensor containing a singlevalue，只有单个值得时候才能用，这个一般在取loss的时候用到，b是基本数据类型，所以肯定是新开辟内存，而且没有梯度这一说了。
c = a.numpy() 这种方式Tensor变量和numpy.array变量是共享底层存储的，注意只有当data.requires_grad=False (老版本pytorch中requires_grad=True时候其实是Variable数据类型) 的时候才有这个method。

评价指标

Words count in article: 1.7k

不同任务的评价指标不一样，每个指标都有其适用的场景。

分类任务

二分类

混淆矩阵(confusion matrix)是最基本的预测结果展示，其他指标都是定义在混淆矩阵上。下图是二分类的混淆矩阵，行代表真实情况，列表示预测情况。P表示Positive(正例)，N表示Negtive(负例)；以TP为例，其中第二个字母表示预测情况(Positive/Negtive)，第一个字母表示我们对预测结果的评价(True)/False).

基于混淆矩阵定义的指标有两个阵营：

第一个阵营的评价指标是偏向于关注positive类, 包括Precision, Recall以及F1.
精确率(Precision)：$P=\frac{TP}{TP+FP}$ . 表示在所有预测为P的样本中，有多少真的是P
召回率(Recall): $R=\frac{TP}{TP+FN}$ . 表示真正为P的样本中，有多少被检测出来
（在恶意检测场景下，通常我们把恶意样本称为正例，把正常样本称为负例）
F1-Score: $F1 = \frac{2 \times P \times R}{P + R}$. F1值定义为召回率和精确率调和平均的2倍（之所以2倍是为了让F1的值域为$[0-1]$）。F1为什么使用调和平均而不是算术平均？调和平均强调了较小值得重要性，比如当召回率接近0，精确率接近1，这其实是很 糟糕的情况，算术平均是0.5， 而调和平均接近0。

第二个阵营的评价指标没有说是更关注某一个类。这个阵营可以划分两个更小的部分，
第一部分是accuracy
准确率(accuracy): $acc = \frac{TP + TN}{ TP + TN + FP +FN} = \frac{T}{T+F}$

第二部分是ROC相关的一些指标
真正例率(True Positive Rate, TPR): $TPR = \frac{TP}{TP+FN}$ TPR是越高越好
假正例率(False Positive Rate, FPR): $FPR = \frac{FP}{FP + TN}$ FPR是越低越好
注意到TPR和FPR的分母是某一行加起来，所以说TPR是从positive类的视角来评价，FPR是从negative类的视角来评价。因此TPR和FPR一起来评价模型的话就是平等对待所有的类。
虽然这些指标都是定义在混淆矩阵上，但是如果我们把分类的阈值调整的话，混淆矩阵是会变的。比如说模型的输出是预测概率，一般情况概率大于0.5判断为正类，概率小于0.5判断为负类；但是如果我们把阈值定位0.4的话，混淆矩阵就会变动了。参考《机器学习》p33 对为什么引入ROC的意义说得很清楚，能从排序本身的质量好坏来评价。我们通过调整阈值，得到不同的混淆矩阵，进而得到不同的(fpr, tpr)，【这里注意下虽然其他指标都会随着混淆矩阵的变动而改变，但是通常我们从TPR和FPR来考虑ROC曲线，比如Precision-Recall曲线也是存在的，但是好像没见过哪里会使用过】。 在得到ROC的时候，通常会把fpr和tpr用小写。
ROC曲线(Receiver Operating Characteristic Curve)就指的是以fpr为横轴，tpr为纵轴得到的一个曲线。
AUC(Aera Under Curve)则表示ROC曲线之下的面积。area under curve好像是在泛指曲线下的面积，但是默认情况下它就是说的area under ROC curve。
很多时候我们把TPR, FPR，ROC，AUC这些都统称为ROC相关指标。

问题1：F1-score和accuracy的区别？
从定义式可以看出准确率只关注正确与否，对真实情况中的每个类都一视同仁；而F1值以精确率和召回率为基础，精确率和召回率都是围绕着TP，所以F1的关注对象是正类这一个类。比如说，二分类问题的两个类分别是A和B，我们可以为A类计算F1值(或召回率、精确率)，也可以给B类计算F1值(或召回率、精确率)；准确率则不存在说是给某个类计算的，它是一个整体的标准。
适用场景：如果每个类都同等重要，那么可以选用ACC，ACC计算也简单，比如在图像分类中ACC就用得多；如果我们更关注某个类或者存在数据不平衡的情况下，用F1值较好，在异常检测等场景用F1就比较多。

问题2：Pricision, Recall 和 TPR, FPR 几个术语的区分？使用场景？：

  1. 术语区分
    • Recall和TPR的定义完全一样，是一个意思；
    • Recall(TPR), FPR是从行的角度统计，分母是某一行加起来，分子是该行中的特定列；
    • Pricision是从列的角度统计，分母是第一列加起来，分子是该列的第一行；
    • Pricision和Recall以及F1通常一起出现；TPR和FPR以及ROC常常一块出现。
  2. 从定义可以看出，精确率和召回率都是围绕着TP，其关注对象是正类这一个类。真正例率TPR, 假正例率FPR则分别关注正类和负类，所以所TPR和FPR相对来说更是一个整体的指标，和acc类似。但是TPR和FPR并且不会因为数据不平衡造成误差。对问题1中的混淆矩阵$TPR = 0.5, FPR = 0.005$， 这个结果是TPR很差，但是FPR很好，这个评价是很恰当的。
    适用场景，如果更加关注正类，那么用recall+precision；如果每个类都重要，那么考虑用TPR+FPR. 如果说要从排序本身质量来评价的话，选后者计算AUC随最合适的，所以很多比赛的评价都是AUC，但其实没必要追求这么好。

多分类

虽然F1可以推广到多分类的情况（macro-F1或者micro-F1），但是这样用的很少；因为既然是多分类，那么一般来说每个类都同等重要，很少有数据不平衡的现象。对于多分类问题直接用准确率最好，计算简单。有些论文里面会用错误率， 错误率 = 1 - 准确率.

维度问题

一般来说，我们希望特征表示向量维度不要太高，因为高维情形下存在“维数灾难”问题，并且高维特征中往往很多是冗余的。但是如果是稀疏的高维表示又存在好处：高维稀疏表示通常会使得问题线性可分，SVM的核技巧就是在做升维变换。所有有时我们需要将维度从大变小，有时我们需要将维度从小变大。

1. 维度从大到小

1.1 维数灾难(curse of dimensionality)

参考文献： 《PR&ML》1.4 pp34
维数灾难指的是在高维情况下出现的数据样本稀疏（特征空间中的样本密度低）、距离计算困难等问题。解决维数灾难的两个重要途径1：1. 降维； 2. 特征选择。 两种方法导致的结果其实都是减小特征向量的维度。注意特征选择的原因除了解决维度灾难，还有去掉与任务无关（冗余）特征的作用。

1.2 降维

降维是无监督的

1.2.1 主成分分析(Principal Component Analysis, PCA)

参考：通俗理解PCA降维作用, 图文并茂的PCA教程
PCA是最经典的无监督降维方法。上面两篇博客对PCA介绍得非常详细。这里只简单列出PCA的步骤。
基本问题：已知数据集有m个数据，每个数据用d维的向量表示，整个数据集表示为$d \times m$的矩阵$\boldsymbol X$.(注意这里$\boldsymbol X$的每一列代表样本，和常用的设计矩阵稍微不同)
目标：我们希望得到一个$d \times d^{‘} $的转换矩阵$\boldsymbol W$, 通过$\boldsymbol X^{‘} = \boldsymbol W \boldsymbol X$就将每个样本表示成$d^{‘}$的向量了。
计算$\boldsymbol W$:

  1. 对样本做中心化处理，即将每个特征维度的均值变为0，这样后面计算协方差的时候更方便
  2. 计算协方差矩阵$\boldsymbol A = \boldsymbol X \boldsymbol X^{T}$，$\boldsymbol A$是$d \times d$的矩阵，其中$ {\boldsymbol A}_{ij}$ 表示特征i与特征j之间的协方差
  3. 对协方差矩阵$\boldsymbol A$做特征分解，即计算特征值和特征向量
  4. 对特征值排序，取最大的$d^{‘}$个特征值对应的特征向量构成转换矩阵$\boldsymbol W$.

1.3 特征选择

特征选择有两个原因：1. 减轻维数灾难问题；2.去除不相关特征往往会降低学习任务的难度。特征选择比较典型的做法是分阶段：第一步先用特征选择过程对初始特征进行“过滤”，第二步再用过滤后的特征来训练模型。还有一种间接的特征选择方法是通过正则化影响优化过程，使得学习到稀疏解，稀疏解中0对应的特征就可以看做是被过滤掉的无用特征。

1.3.1 分阶段

先特征选择，然后训练模型是最典型机器学习方式。简单来说，就是从一大堆特征中选择最重要的一些特征。什么特征是重要的呢？我们人为指定评价标准。通常来说是设计一个相关统计量来衡量某个特征与任务的相关程度，然后选择出相关统计量最大的k个特征。我们可以同时独立地选择k个特征，也可以以贪心算法迭代挑选出k个特征。

1.3.1.1 同时独立选择k个特征

比如sklearn中的特征选择模块中的SelectKBest和卡方检验chi2配合使用就是典型的例子。卡方检验的得分就是我们定义的相关统计量，然后根据这个相关统计量选择得分最高的top-k个特征。除了卡方检验，常用的相关统计量还有互信息熵 mutual_info等。

1.3.1.2 贪心迭代挑选k个特征

类似决策树算法，参考《机器学习》p248.

1.3.2 稀疏解

参考《机器学习》p252.
L1正则化可以鼓励优化算法得到模型的稀疏解2 。 以线性模型为例，参数$ \boldsymbol w $取得稀疏解以为这初始的$d$个特征中仅有对应着$ \boldsymbol w $ 的非零分量的特征才会出现在最终的模型中，这其实就是间接完成了特征选择目的。这里的稀疏解也可以成为参数稀疏，注意参数稀疏和特征稀疏（稀疏表示）不是一回事，前者的对象是参数向量，后者的对象是特征表示向量。
有篇博客稀疏性的优点总结了参数稀疏的优点有：1. 起到特征选择的作用， 2. 可解释。但这篇博客混淆了参数稀疏和特征稀疏， 线性可分和稀疏存储是特征稀疏带来的好处。

2. 维度从小到大

参考《机器学习》p255.
稀疏表示的好处简单来说就是：维度越高则线性可分的概率越高。而且稀疏表示并不会造成存储或计算上的巨大负担，因为稀疏矩阵有很多高效的存储方法和计算加速。
稀疏表示具有上述优势，所以我们希望把样本表示成稀疏形式。稀疏表示是我们的目的，而高维往往是稀疏的必要条件，所以我们才冒着“维度灾难”的风险升维。把原始低纬的特征映射成高维稀疏特征的过程叫做稀疏编码或字典学习。
SVM的核方法就是这个思路3，对于在原始空间内线性不可分的情况，可以将样本从原始空间映射到一个更高维的特征空间，在这个高维特征中间内是线性可分的。比如高斯核所代表的映射就是讲原始空间映射到无限维的希尔伯特空间，理论上来说无限维那肯定是相当的稀疏，妥妥的线性可分。当然无限维中的计算是维度灾难，而核技巧通过定义核函数巧妙地绕过了计算问题。

核函数

问题1：如何理解kernel？

核函数$K(x, z)$其实是在度量样本$x$和样本$z$之间相似性。

常用的核函数有一下几种：

  1. 高斯核函数RBF

    如果我们把$x-z$看做一个距离变量$d$，并且忽略高斯分布的归一化部分，则高斯核可以写成

    这能更直地的看出高斯核其实就是以距离$d$为变量的高斯分布($\mu=0$, $\delta$人工指定)。所以当$x$和$z$的欧几里得距离很小时，对应的高斯核响应很大。

  2. 线性核

    用线性核其实就跟不用核函数是一样的，在svm中算是鸡肋。这里把 $x^T z$ 看作是两者之间的相似度度量，在Word2vec的训练中其实就是用的这个内积来度量两个单词的相似性。

  3. 多项式核

给定核函数，其实会通过在样本两两之间得到一个相似度的输出。假设数据集中一共有$n$个样本，所以最后得到的会是一个$n \times n$维的矩阵, 通常称之为核矩阵。其实，如果我们有自定义的衡量样本之间相似度的方法（比如说网络节点之间用边的权重直接就是节点间相似度了），其实可以直接给出核矩阵。 特别注意核矩阵是半正定矩阵，既然每个元素都是相似度，那么相似度应该是要$\geq 0$的。

主要考虑几个方面的超参数：

  1. 指定kernel函数。

    首先是指定用哪种kernel，其次要设置kernel的参数。注意sklearn是按照libsvm文献里面的kernel公式，通常书本上说的是简化形式。

    线性核：kernel='linear'

    多项式核：kernel='poly', 需要指定多项式的次数$d$: degree=3，也可以指定gamma和coef0, 因为libsvm中公式为$k(x,z) = (\gamma x^T z + r)^d$更完整

    高斯核：kernel='rbf', 需要指定带宽gamma=1e-3（默认gamma等于$\frac{1}{特征个数}$）, 注意这里 $\gamma = \frac{1}{2\delta^2}$, 有些文献里面高斯核公式里面用的$\gamma$而不是$\delta$，其实是一个意思，都是用来限制带宽。$\gamma$越大，$\delta$越小，带宽越小，支持向量的作用范围越小，（？？最后得到的支持向量个数越少？？）

  2. 指定对软间隔错误损失的容忍度$C$. C=1e-3

    参考《机器学习》p130.

    优化目标：$\min \limits_{w,b} \frac{1}{2}||w||^2 + C \sum^n_{i=1} loss(f(\mathbf x_i), y_i)$

    注意上面的公式调节正则化项和经验风险的系数$C$是放在经验风险这一项的

    在设计损失函数更多时候是 $损失函数 = 经验风险 + \lambda 正则化项$

  3. 控制训练的过程。比如说迭代次数，内存占用等一般不需要不需要调节

损失函数

注意：损失函数和评价指标(性能度量)不是一回事儿
损失函数(loss function)和代价函数(cost function)一个意思；风险函数(risk function)可以看做是在数据集平均意义下损失函数的期望，其实可以不用很严格的区分。经验风险(empirical risk)和经验损失(empirical loss)通常混用，指的是在训练集的平均损失。 参考：《统计机器学习》p7-8

交叉熵损失函数
交叉熵与softmax不是一回事。交叉熵可以得出两个概率分布的距离，其输入为两个概率分布。经过one-hot编码的标签向量可以看做一个概率分布，而softmax是输出恰好可以看做是一个概率分布。所以交叉熵和softmax经常出现在一块，交叉熵作为一种距离度量，是属于损失函数；而softmax属于一种特殊的激活函数。 参考：《深度学习》 3.1

总结

设计损失函数的基本套路就是用训练集上的经验风险来代替真实的数据分布风险，训练集上的风险函数也不好求，所以就在训练集上用代理损失函数。代理损失函数其实不光数学性质好，往往也能提高训练效果。
