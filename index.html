评价指标

不同任务的评价指标不一样，每个指标都有其适用的场景。

分类问题

二分类

混淆矩阵(confusion matrix)是最基本的预测结果展示，其他指标都是定义在混淆矩阵上。下图是二分类的混淆矩阵，行代表真实情况，列表示预测情况。P表示Positive(正例)，N表示Negtive(负例)；以TP为例，其中第二个字母表示预测情况(Positive/Negtive)，第一个字母表示我们对预测结果的评价(True)/False).

基于混淆矩阵定义的指标
精确率(Precision)：$P=\frac{TP}{TP+FP}$ . 表示在所有预测为P的样本中，有多少真的是P
召回率(Recall): $R=\frac{TP}{TP+FN}$ . 表示真正为P的样本中，有多少被检测出来
（在恶意检测场景下，通常我们把恶意样本称为正例，把正常样本称为负例）
F1-Score: $F1 = \frac{2 \times P \times R}{P + R}$. F1值定义为召回率和精确率调和平均的2倍（之所以2倍是为了让F1的值域为$[0-1]$）。F1为什么使用调和平均而不是算术平均？调和平均强调了较小值得重要性，比如当召回率接近0，精确率接近1，这其实是很 糟糕的情况，算术平均是0.5， 而调和平均接近0。

问题：F1-score和accuracy(准确率)的区别：准确率的定义为$acc = \frac{TP + TN}{ TP + TN + FP +FN} = \frac{T}{T+F}$，从定义式可以看出准确率只关注正确与否，对真实情况中的每个类都一视同仁；而F1值以精确率和召回率为基础，精确率和召回率都是围绕着TP，所以F1的关注对象是正类这一个类。比如说，二分类问题的两个类分别是A和B，我们可以为A类计算F1值(或召回率、精确率)，也可以给B类计算F1值(或召回率、精确率)；准确率则不存在说是给某个类计算的，它是一个整体的标准。适用场景：如果每个类都同等重要，那么可以选用ACC，ACC计算也简单，比如在图像分类中ACC就用得多；如果我们更关注某个类或者存在数据不平衡的情况下，用F1值较好，在异常检测等场景用F1就比较多。
举例说明：有如下混淆矩阵

明显数据不平衡，这在异常检测中很常见，把异常定义为正例，那异常样本的数量本来就少。这个混淆矩阵算出来$acc= \frac{1000}{1010} = 0.99$，简直不要太好；而$F1 = 0.5$。 显然这种情况用F1会更合适。

问题：Pricision, Recall 和 TPR, FPR 几个术语的区分：
真正例率(True Positive Rate, TPR): $TPR = \frac{TP}{TP+FN}$
假正例率(False Positive Rate, FPR): $FPR = \frac{FP}{FP + TN}$

  • Recall和TPR的定义完全一样，是一个意思；
  • Recall(TPR), FPR是从行的角度统计，分母是某一行加起来，分子是该行中的特定列；
  • Pricision是从列的角度统计，分母是第一列加起来，分子是该列的第一行；
  • Pricision和Recall以及F1通常一起出现；TPR和FPR以及ROC常常一块出现。

多分类

虽然F1可以推广到多分类的情况（macro-F1或者micro-F1），但是这样用的很少；因为既然是多分类，那么一般来说每个类都同等重要，很少有数据不平衡的现象。对于多分类问题直接用准确率最好，计算简单。有些论文里面会用错误率， 错误率 = 1 - 准确率.

维度问题

一般来说，我们希望特征表示向量维度不要太高，因为高维情形下存在“维数灾难”问题，并且高维特征中往往很多是冗余的。但是如果是稀疏的高维表示又存在好处：高维稀疏表示通常会使得问题线性可分，SVM的核技巧就是在做升维变换。所有有时我们需要将维度从大变小，有时我们需要将维度从小变大。

1. 维度从大到小

1.1 维数灾难(curse of dimensionality)

参考文献： 《PR&ML》1.4 pp34
维数灾难指的是在高维情况下出现的数据样本稀疏（特征空间中的样本密度低）、距离计算困难等问题。解决维数灾难的两个重要途径1：1. 降维； 2. 特征选择。 两种方法导致的结果其实都是减小特征向量的维度。注意特征选择的原因除了解决维度灾难，还有去掉与任务无关（冗余）特征的作用。

1.2 降维

降维是无监督的

1.2.1 主成分分析(Principal Component Analysis, PCA)

参考：通俗理解PCA降维作用, 图文并茂的PCA教程
PCA是最经典的无监督降维方法。上面两篇博客对PCA介绍得非常详细。这里只简单列出PCA的步骤。
基本问题：已知数据集有m个数据，每个数据用d维的向量表示，整个数据集表示为$d \times m$的矩阵$\boldsymbol X$.(注意这里$\boldsymbol X$的每一列代表样本，和常用的设计矩阵稍微不同)
目标：我们希望得到一个$d \times d^{‘} $的转换矩阵$\boldsymbol W$, 通过$\boldsymbol X = \boldsymbol W \boldsymbol X$就将每个样本表示成$d^{‘}$的向量了。
计算$\boldsymbol W$:

  1. 对样本做中心化处理，即将每个特征维度的均值变为0，这样后面计算协方差的时候更方便
  2. 计算协方差矩阵$\boldsymbol A = \boldsymbol X \boldsymbol X^{T}$，$\boldsymbol A$是$d \times d$的矩阵，其中$ {\boldsymbol A}_{ij}$ 表示特征i与特征j之间的协方差
  3. 对协方差矩阵$\boldsymbol A$做特征分解，即计算特征值和特征向量
  4. 对特征值排序，取最大的$d^{‘}$个特征值对应的特征向量构成转换矩阵$\boldsymbol W$.

1.3 特征选择

特征选择有两个原因：1. 减轻维数灾难问题；2.去除不相关特征往往会降低学习任务的难度。特征选择比较典型的做法是分阶段：第一步先用特征选择过程对初始特征进行“过滤”，第二步再用过滤后的特征来训练模型。还有一种间接的特征选择方法是通过正则化影响优化过程，使得学习到稀疏解，稀疏解中0对应的特征就可以看做是被过滤掉的无用特征。

1.3.1 分阶段

先特征选择，然后训练模型是最典型机器学习方式。简单来说，就是从一大堆特征中选择最重要的一些特征。什么特征是重要的呢？我们人为指定评价标准。通常来说是设计一个相关统计量来衡量某个特征与任务的相关程度，然后选择出相关统计量最大的k个特征。我们可以同时独立地选择k个特征，也可以以贪心算法迭代挑选出k个特征。

1.3.1.1 同时独立选择k个特征

比如sklearn中的特征选择模块中的SelectKBest和卡方检验chi2配合使用就是典型的例子。卡方检验的得分就是我们定义的相关统计量，然后根据这个相关统计量选择得分最高的top-k个特征。除了卡方检验，常用的相关统计量还有互信息熵 mutual_info等。

1.3.1.2 贪心迭代挑选k个特征

类似决策树算法，参考《机器学习》p248.

1.3.2 稀疏解

参考《机器学习》p252.
L1正则化可以鼓励优化算法得到模型的稀疏解2 。 以线性模型为例，参数$ \boldsymbol w $取得稀疏解以为这初始的$d$个特征中仅有对应着$ \boldsymbol w $ 的非零分量的特征才会出现在最终的模型中，这其实就是间接完成了特征选择目的。这里的稀疏解也可以成为参数稀疏，注意参数稀疏和特征稀疏（稀疏表示）不是一回事，前者的对象是参数向量，后者的对象是特征表示向量。
有篇博客稀疏性的优点总结了参数稀疏的优点有：1. 起到特征选择的作用， 2. 可解释。但这篇博客混淆了参数稀疏和特征稀疏， 线性可分和稀疏存储是特征稀疏带来的好处。

2. 维度从小到大

参考《机器学习》p255.
稀疏表示的好处简单来说就是：维度越高则线性可分的概率越高。而且稀疏表示并不会造成存储或计算上的巨大负担，因为稀疏矩阵有很多高效的存储方法和计算加速。
稀疏表示具有上述优势，所以我们希望把样本表示成稀疏形式。稀疏表示是我们的目的，而高维往往是稀疏的必要条件，所以我们才冒着“维度灾难”的风险升维。把原始低纬的特征映射成高维稀疏特征的过程叫做稀疏编码或字典学习。
SVM的核方法就是这个思路3，对于在原始空间内线性不可分的情况，可以将样本从原始空间映射到一个更高维的特征空间，在这个高维特征中间内是线性可分的。比如高斯核所代表的映射就是讲原始空间映射到无限维的希尔伯特空间，理论上来说无限维那肯定是相当的稀疏，妥妥的线性可分。当然无限维中的计算是维度灾难，而核技巧通过定义核函数巧妙地绕过了计算问题。

基本问题：已知包含m个样本的数据集$\mathbb X = \{ x_1, x_2,…,x_m \}$， 每个样本从真实数据分布$p_{data}(x)$中独立采样得到，现在我们希望用$p(x, \theta)$来拟合真实分布，需要做的事情就是估计参数$\theta​$。

要清楚我们的求解对象是$\theta​$, 估计一个概率模型的参数的方法有很多种，最常用的是最大似然估计(Maximum Likelihood Estimation, MLE)和最大后验估计(MAximum Posteriori estimation, MAP).

最大后验估计

最大后验估计是从贝叶斯估计推导出来的，但是不能完全等同于贝叶斯估计，可以把最大后验估看做是贝叶斯估计的简化实用版本。

频率派的视角是真实参数$\theta$是未知的定值。而数据集是随机采样得到，是随机的。贝叶斯统计的视角认为数据集能够被直接观测到，因此不是随机的；反而是真实参数$\theta​$是未知或不确定的，因此可以表示成随机变量。

对本文开头的基本问题，贝叶斯估计不是给$\theta​$赋某个值，而是给出$\theta​$的概率分布，理由很简单，因为贝叶斯认为$\theta​$是个随机变量，那自然是用概率分布来描述随机变量。贝叶斯估计$\theta​$ 首先需要人为设定$\theta​$ 的先验(prior)概率分布 $p_{prior}(\theta)​$, 然后在观察数据集的条件下用贝叶斯公式得到$\theta​$的后验(poster)概率分布$p_{poster}​$，后验概率就看做是贝叶斯估计求解出来的$\theta​$.

multiprocessing

python多进程库multiprocessing的优点：The multiprocessing package offers both local and remote concurrency, effectively side-stepping the Global Interpreter Lock by using subprocesses instead of threads.

创建进程

在multiprocessing , 用Process对象来表示一个进程，processes are spawned by creating a Process object and then calling its start() method.

一般来说用到多进程的地方我们都希望创建比较多的子进程，所以进程池Pool是创建子进程的推荐用法。One can create a pool of processes which will carry out tasks submitted to it.

第一步：创建进程池

第二步：向进程池中添加任务
<第一种> 一次添加一个任务
pool.apply_async(func[, args[, kwds[, callback[, error_callback]]]])
<第二种>map自动分割数据，然后添加多个任务
ret = pool.map(func, iterable[, chunksize])
This method chops the iterable into a number of chunks which it submits to the process pool as separate tasks. 这种方法会自动将每个子进程的结果reduce一下，很好用。
如果数据很大的话，可以使用迭代器版本的pool.imap(func, iterable[, chunksize])，注意imap的chunksize默认为1，一般我们设置大一些更高效。
map_async(func, iterable[, chunksize[, callback[, error_callback]]])
